3 - variables

Variables allow us to assign data to a word, then we can use that word within our program instead of the data. Then, if the variable's value changes we only have to change the variable's value instead of re-writing the entire program.

declaration description 
        var myName = 'Arya';
        console.log(myName);
        // Output: Arya
  • var, short for variable, is the JavaScript keyword that will create a new variable for us.
  • myName is chosen by a developer. Notice that the word has no spaces, and each new word is capitalized. This is a common convention in JavaScript, and is called camelCase.
  • = means to assign whatever's next to the variable.
  • 'Arya' is the value that the equals = assigns into the variable myName. 
        var strength = '50,000 pounds';
        var age = 46;
        var hasPet = true;
        console.log(hasPet);

Variables can hold any data type, like strings, numbers, and Booleans. They can also hold data types like arrays, functions and objects

 
     var weatherCondition = 'Monday: Raining cats and dogs';
     weatherCondition = 'Tuesday: Sunny';

     console.log(weatherCondition); 
     // Output: 'Tuesday: Sunny'

Variables are useful in two ways:

  • they allow us to use the same value over and over, without having to write a string or other data type over and over.
  • More importantly, we can assign variables different values that can be read and changed by the program without altering our code.

We can change a variable's value if we want

We created a variable by using the keyword var and the name weatherCondition.

Then, we took the existing weatherCondition variable, and set its value equal to 'Tuesday: Sunny'.

 
    var myPet = 'armadillo';
    console.log('I own a pet ' + myPet + '.'); 
    // Output: 'I own a pet armadillo.'

It is also possible to put a variable's value into a string.

he JavaScript term for this idea is interpolation.

We can use the + operator to interpolate (insert) a variable into a string

5 - Arrays

arrays are used to store multiple values in a single variable.

Variables hold 1 values. However Arrays can hold multiple values at once.

An array can hold many values under a single name, and you can access the values by referring to an index number.

description declaration

best is to write the array in 1 line. This is the preferred method

var cars = ["Saab", "Volvo", "BMW"];

Access an element of the array:

You refer to an array element by referring to the index number. in other words, the input order of the array will set the index number of each value.

Array indexes start with 0.

In this example, saab has inder order [0] but the second line modifies the first element in var cars and thus the response is opel

 
          var cars = ["Saab", "Volvo", "BMW"];
          var cars [0] = "Opel"
          document.getElementById("demo").innerHTML = cars[0];

Access the Full Array. the full array can be accessed by referring to the array name.

 
var cars = ["Saab", "Volvo", "BMW"];
        document.getElementById("demo").innerHTML = cars;

Arrays are Objects. Arrays use numbers to access its "elements". use objects when you want the element names to be strings (text). use arrays when you want the element names to be numbers.

In this example, person[0] returns John

        var person = ["John", "Doe", 46];
        document.getElementById("demo").innerHTML = person[0];

Arrays are Objects. Objects use names to access its "members".

In this example, person.firstName returns John. note the curly brackets

        var person = {firstName:"John", lastName:"Doe", age:46};
        document.getElementById("demo").innerHTML = person["firstName"];

6 - comments

JavaScript comments can be used to explain JavaScript code, and to make it more readable.

JavaScript comments can also be used to prevent execution, when testing alternative code.

DESCRIPTION EXAMPLE

Single Line Comments. Single line comments start with //. Any text between // and the end of the line will be ignored by JavaScript (will not be executed).

/td>

// console.log ('this message will not show up as it is uncommented by the single line comment at THE BEGINNING of the line.').

Use a single line comment at the end of each line to explain the code.

Multi-line comments start with /* and end with */. Any text between /* and */ will be ignored by JavaScript.

7 - math operators

JavaScript includes the general math operators that you can find on a calculator

  1. Add: +
  2. Subtract: -
  3. Multiply: *
  4. Divide: /
 
          console.log(46+3.5)
          console.log(2016-1969)
          console.log(65/240)
          console.log(0.2708333333333333*100)

modulus: %

The idea behind the modulus is to show you the remainder after you divide a number.

So, if you divide 13 / 5, 5 goes into 13 two times, and there will be 3 remaining. A modulus, denoted by a %, would take 13 % 5 and return the remainder 3.
